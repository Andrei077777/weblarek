# Интернет-магазин «Web-Larёk»

«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component

Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`

#### Класс Api

Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter

Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` - хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Модели данных

#### Класс Products

1.Класс Products, который представляет собой модель данных для работы с каталогом товаров в приложении.

items: Это приватный массив, где хранятся все товары, полученные с сервера или загруженные вручную.
selectedProduct: Это приватная переменная для хранения одного конкретного товара, который пользователь выбрал для просмотра в деталях. Изначально он равен null, так как ничего не выбрано.
api: Это приватная ссылка на объект API-клиента. Она передается в конструктор извне.

2.Конструктор:

constructor(api: IApi) {
this.api = api;
}
Конструктор принимает один обязательный аргумент — api.

3.Методы для работы с данными:

3.1 Загрузка данных с сервера
async loadItems(): Promise<void> {
  const data = await this.api.get<{ total: number; items: IProduct[] }>('/product');
  this.items = data.items;
}

Он вызывает this.api.get('/product'). Тип <{ total: number; items: IProduct[] }> подсказывает TypeScript, какой структуры данных ожидать от сервера.
Он ждет (await) ответа от сервера.
Получив ответ, он берет массив товаров (data.items) и сохраняет его в свое внутреннее поле this.items.

3.2 Установка данных напрямую (для тестов)
setItems(items: IProduct[]): void {
  this.items = items;
}
Этот метод позволяет заполнить модель товарами без обращения к серверу

3.3 Получение данных
getItems(): IProduct[] {
  return this.items;
}
Простой метод-геттер, который возвращает копию массива всех товаров.

getItemById(id: string): IProduct | undefined {
  return this.items.find(item => item.id === id);
}
Метод для поиска одного товара по его id. Он использует встроенный метод массива .find(). Если товар найден, он возвращается; если нет — возвращается undefined.

3.4 Работа с выбранным товаром
setSelectedProduct(product: IProduct): void {
  this.selectedProduct = product;
}
getSelectedProduct(): IProduct | null {
  return this.selectedProduct;
}
Эти два метода управляют состоянием "выбранного товара". Когда пользователь кликает на карточку товара, вызывается setSelectedProduct(). А когда нужно отобразить модальное окно с деталями, компонент вызывает getSelectedProduct(), чтобы получить данные для рендера.

#### Класс Cart

Класс Cart (Корзина), который является моделью данных для управления покупками пользователя. Он отвечает за хранение списка выбранных товаров и выполнение операций с этим списком.

1.
private items: IProduct[] = [];
private api: IApi;
items: Приватный массив, в котором хранятся объекты товаров, добавленных в корзину.
api: Приватная ссылка на API-клиент, которая передается через конструктор.

2. Конструктор
constructor(api: IApi) {
  this.api = api;
}
Класс не создает свой собственный API-клиент, а принимает его извне.

3. Методы управления корзиной

3.1.Добавление товара 

async addItem(product: IProduct): Promise<void> {
  if (!this.isInCart(product.id)) {
    this.items.push(product);
    // Здесь должен быть запрос к серверу для синхронизации
    await Promise.resolve(); // Заглушка вместо реального запроса
  }
}
Метод принимает объект product.
Проверка: Перед добавлением он проверяет, нет ли уже товара с таким id в корзине (!this.isInCart(product.id)), чтобы избежать дубликатов.
Действие: Если товара нет, он добавляется в массив items.

3.2.Удаление товара 

async removeItem(id: string): Promise<void> {
  this.items = this.items.filter(item => item.id !== id);
  // Здесь должен быть запрос к серверу для синхронизации
  await Promise.resolve(); // Заглушка вместо реального запроса
}
Метод принимает id товара, который нужно удалить.
Действие: Он не ищет и не удаляет элемент напрямую. Вместо этого он создает новый массив, в который входят только те товары, чей id не совпадает с переданным.

3.3.Очистка корзины 

clear(): void {
  this.items = [];
}
Просто сбрасывает массив товаров в пустой. В реальном приложении здесь также был бы вызов API.

4. Методы получения информации (Геттеры)
Эти методы не изменяют состояние корзины, а лишь предоставляют данные о ней.

Подсчет количества 

getCount(): number {
  return this.items.length;
}
Возвращает длину массива items. Это число используется для отображения счетчика товаров на иконке корзины.

Расчет общей стоимости 

getTotalPrice(): number {
  return this.items.reduce((total, item) => total + (item.price || 0), 0);
}
Использует метод .reduce(), чтобы пройтись по всем товарам и сложить их цены.

Проверка наличия товара 

isInCart(id: string): boolean {
  return this.items.some(item => item.id === id);
}

Вспомогательный метод, который возвращает true, если в корзине есть товар с указанным id. Он используется внутри addItem для проверки на дубликаты.


#### Класс Cart

Этот код описывает класс Buyer (Покупатель), который является моделью данных для хранения и управления информацией о клиенте, оформляющем заказ. 
1.
private data: IBuyer = { ... };
private api: IApi;
data: Приватное поле, в котором хранится объект с данными покупателя. Он инициализируется "пустыми" значениями по умолчанию.
api: Приватная ссылка на API-клиент, которая передается через конструктор для отправки данных на сервер.

2. Конструктор

constructor(api: IApi) {
  this.api = api;
}
Класс принимает готовый объект api извне. Это позволяет использовать один и тот же экземпляр API во всем приложении, что делает архитектуру более гибкой и упрощает тестирование.

3. Методы для изменения данных (Сеттеры)
Класс предоставляет два способа изменения данных:

3.1. Пакетная установка (setData)

setData(data: Partial<IBuyer>): void {
  this.data = { ...this.data, ...data };
}
Этот метод принимает объект, который может содержать только часть свойств IBuyer. Он объединяет текущий объект this.data с новыми данными. 

3.2. Индивидуальные сеттеры

setPayment(payment: TPayment): void { ... }
setEmail(email: string): void { ... }
// и так далее
Эти методы позволяют изменять каждое поле по отдельности. 

4. Методы для получения данных (Геттеры)
4.1
getData()

getData(): IBuyer {
  return this.data;
}
Просто возвращает копию текущих данных покупателя. 

4.2
clear()

typescript
Копировать
clear(): void {
  this.data = { /* начальные значения */ };
}
Сбрасывает все поля данных к их начальному состоянию. Полезно после успешного оформления заказа, чтобы подготовить форму для следующего покупателя.

5. Валидация данных (validate)

validate(): { [key in keyof IBuyer]?: string } { ... }
Он проверяет, все ли обязательные поля заполнены корректно.

Создается пустой объект errors.
Для каждого поля (payment, address, email, phone) выполняется проверка.
Если поле пустое (или не выбрано), в объект errors добавляется свойство с именем поля и текстом ошибки.
Результат: Метод возвращает объект. Если он пустой ({}), значит, ошибок нет. Если в нем есть свойства (например, { payment: 'Ошибка' }), значит, валидация не пройдена. 

6. Отправка заказа (submitOrder)

async submitOrder(): Promise<void> {
  await this.api.post<void>('/order', this.data, ApiPostMethods.POST);
  this.clear();
}
Этот метод завершает процесс оформления заказа:

Отправка данных: Он асинхронно отправляет накопленные данные покупателя (this.data) на сервер по адресу /order методом POST.
Очистка: После успешной отправки (когда промис await разрешится) он вызывает метод clear(), чтобы сбросить форму.
